# Claude Design 프롬프트 — 가계부 웹 앱 UI

아래 내용을 Claude Design에 붙여넣으면 됩니다.

---

개인용 가계부(household budget tracker) 웹 앱의 UI를 디자인해줘. React + TypeScript로 구현될 프론트엔드이고, 데스크톱 우선이되 반응형으로 동작해야 해.

## 제품 성격
- 개인이 매일 쓰는 도구. 빠른 입력과 한눈에 파악이 가장 중요하다.
- 따뜻하고 정돈된 느낌. 금융 앱이지만 차갑지 않고, 장부·영수증의 친숙함을 살린 톤.
- 한국어 UI, 원화(₩) 기준.

## 디자인 톤
- 밝고 깔끔하며 여백이 넉넉한 레이아웃
- 숫자 가독성이 핵심 (금액, 합계가 잘 읽혀야 함)
- 카테고리별 색상 시스템 (식비/교통/주거 등 구분)
- 과한 장식 없이, 데이터가 주인공

## 필요한 화면
1. **대시보드** — 들어가면 자동 요약이 바로 보이는 화면. 이번 달 총수입/총지출/잔액, 상위 지출 카테고리 랭킹, 자주 가는 가맹점 TOP, 지난달 대비 증감(+/− %), 평소 대비 튀는 지출 강조, 예산 현황 시각화, 즐겨찾기 필터, 최근 거래
2. **거래 내역** — 날짜별 그룹화된 거래 목록, 검색·필터(기간/카테고리/태그/결제수단), 각 항목 수정·삭제
3. **거래 입력** — 빠른 입력 폼(날짜/금액/유형/카테고리/결제수단/메모), 모달 또는 사이드패널
4. **예산** — 카테고리별 월 예산 대비 사용액 시각화. 알림·팝업 없이 색상으로 상태 표현(여유=초록, 임박=노랑, 초과=빨강). 진행바 또는 더 나은 시각화
5. **고정 지출** — 자주 쓰는 반복 항목(월세·구독료 등)을 템플릿으로 등록한 목록. 각 템플릿을 누르면 오늘 날짜 거래로 빠르게 추가(금액 등 즉석 수정 가능). 등록 폼(이름/금액/카테고리/결제수단/활성여부)
6. **가져오기** — CSV 업로드, 열 매핑 UI(어느 열이 날짜·금액·가맹점인지 지정), 중복 거래 확인
7. **분석** — 카테고리 비중(도넛), 대분류→소분류 드릴다운, 카테고리별 월별 추이, 이번 달 vs 지난 달 비교, 태그 기반 집계(예: 데이트), 기간/카테고리/태그/가맹점 필터로 탐색, 리포트 내보내기

## 핵심 UX 요구
- 거래 입력이 2~3번의 동작으로 끝나야 함 (가장 자주 쓰는 기능)
- 예산은 "실시간 추적"이 아니라 입력된 거래의 합계를 보여주는 개념. 강제 알림 없이 색상 시각화로만 상태 전달
- 카테고리는 대분류–소분류 계층 구조, 자유 태그도 붙일 수 있음
- **태그는 카테고리와 독립된 횡단 축**(예: `데이트` 태그로 외식·영화·선물을 묶어 집계). 거래 입력·목록·분석에서 태그가 자연스럽게 보여야 함
- **거래 추적성**: 각 거래에 원본 가맹점명과 정리된 별칭을 함께 표시(예: "(주)스타벅스코리아 → 스타벅스"). 목록에서 "이게 뭔 지출인지" 바로 알아볼 수 있게
- **가맹점 자동 분류**: CSV 가져오기 화면에서 "이 가맹점을 다음부터 이 카테고리로 자동 분류" 규칙을 만드는 UX 필요
- 결제수단: 현금/체크카드/신용카드/계좌이체/간편결제

## 기본 카테고리(색상 시스템 참고용)
- 식비, 교통, 주거·통신, 생활, 건강, 문화·여가(운동·헬스 포함), 쇼핑, 경조사·기타
- 수입: 급여, 용돈, 이자·투자수익, 환급, 기타수입
- 분류 기준: 물건을 사면 쇼핑, 경험·서비스면 문화·여가

먼저 대시보드부터 디자인하고, 전체적인 색상·타이포·컴포넌트 스타일의 방향을 잡아줘.
